# 🎒 프로그래밍 입문자를 위한 '북어 트립' 아키텍처 학습 가이드

이 학습서에서는 '북어 트립' 프로젝트에 적용된 현대 웹 프로그래밍의 핵심 개념과 파일 구조, 그리고 각 기능들이 어떻게 서로 맞물려 통신하는지 이론적으로 설명합니다. 웹 주니어 개발자로 성장하기 위한 첫걸음으로 아래 원리들을 차근차근 배워 봅시다!

---

## 1. 클라이언트-서버 모델 & HTTP 통신
우리가 사용하는 모든 웹 사이트는 기본적으로 **클라이언트(Client)**와 **서버(Server)**가 통신하는 구조입니다.

```
┌────────────────────────┐                   ┌────────────────────────┐
│  Client (브라우저)      │ ─── HTTP POST ──> │   Server (FastAPI)     │
│  - index.html, CSS, JS │ <── JSON Data ─── │   - api/index.py       │
└────────────────────────┘                   └────────────────────────┘
```

* **클라이언트 (Client)**: 사용자가 직접 눈으로 보고 조작하는 웹 브라우저(`index.html`, `js/app.js`)입니다. 사용자가 입력창에 도시 이름을 쓰면 브라우저는 이를 가공하여 서버로 보내달라는 요청을 작성합니다.
* **서버 (Server)**: 보이지 않는 클라우드 우주 어딘가에서 연산과 정보 가공을 전담하는 컴퓨터(`api/index.py`)입니다.
* **HTTP 통신 (Request & Response)**:
  * **요청 (Request)**: 브라우저가 `fetch()` 함수를 사용하여 서버의 `/api/generate` 주소로 사용자 취향(스타일, 일정, 테마)을 담아 **POST** 방식으로 전달합니다.
  * **응답 (Response)**: 서버는 Gemini AI와 통신하여 팩트 기반 이야기를 조립한 후, 구조화된 **JSON 데이터** 형태로 변환하여 클라이언트에게 다시 응답해 줍니다.

## 2. 이벤트 버블링과 전파 차단 (`event.stopPropagation()`)
웹 화면 개발에서 여러 개의 버튼이나 카드가 겹쳐 있을 때 발생하는 자바스크립트의 핵심 동작 원리입니다.

### ① 이벤트 버블링 (Event Bubbling)
HTML 태그들은 서로를 감싸는 부모-자식 관계를 가집니다. 이때 자식 태그(예: 보관함 카드의 `삭제 X 버튼`)를 클릭하면, 클릭 이벤트가 자식에서 끝나지 않고 부모 태그(예: 보관함 `카드 몸통 전체`)로 마치 물속의 거품(Bubble)처럼 솟아올라 연달아 실행되는 현상입니다.

### ② 문제 발생 및 해결책
만약 버블링을 방지하지 않으면, 사용자가 보관함 카드의 '삭제 버튼'을 눌렀을 때, 책이 삭제되는 동시에 부모 카드의 '책 열기' 함수가 연속으로 작동하여 지워진 책이 화면에 열려 버리는 버그가 생깁니다.
```javascript
// ⚠️ 삭제 단추 클릭 이벤트 핸들러
btnDelete.addEventListener("click", (evt) => {
  evt.stopPropagation(); // 👈 거품처럼 위로 솟아오르는 이벤트를 여기서 차단!
  
  // 책 삭제 로직 실행...
});
```
`evt.stopPropagation()` 한 줄을 적어 줌으로써 삭제 신호가 카드 몸통까지 흘러 들어가지 않도록 물리적으로 차단하여 안정적인 UI를 완성할 수 있습니다.

## 3. CSS 배경 슬라이싱을 이용한 양면 와이드 표지 연출
북어 트립은 책을 딱 펼쳤을 때 양면에 하나의 일러스트 그림이 광활하게 이어지는 **양면 와이드 팝업 디자인**을 보여줍니다. 이는 이미지를 쪼개지 않고 CSS 속성만으로 구현된 영리한 기법입니다.

```
                  [ 1024 x 768 표지 원본 이미지 ]
           ┌────────────────────────┬────────────────────────┐
           │                        │                        │
           │      Left Half (50%)   │    Right Half (50%)    │
           │                        │                        │
           └────────────────────────┴────────────────────────┘
            [Page 1: cover-half-left] [Page 2: cover-half-right]
```

* **슬라이싱 원리**:
  1. 두 개의 독립적인 페이지(Page 1, Page 2)에 **완전히 동일한 표지 이미지**를 배경으로 얹습니다.
  2. 이미지의 가로 크기를 실제 페이지 너비의 **200%**로 확대합니다 (`background-size: 200% 100%`).
  3. 왼쪽 페이지에는 이미지의 왼쪽 반만 보여주고 (`background-position: left center`), 오른쪽 페이지에는 오른쪽 반만 보여주어 (`background-position: right center`) 책을 폈을 때 두 이미지가 정교하게 이어지도록 만듭니다.

## 4. 로컬 캐싱을 이용한 중복 요청 방지 (LocalStorage Cache)
네트워크 연결은 비용이 발생하고 예기치 못한 실패가 생길 수 있습니다. 이를 효율적으로 해결하기 위해 **로컬 스토리지 캐시(Local Cache)** 기술을 도입했습니다.

* **로컬 캐시 매커니즘**:
  책을 생성하기 전 로컬스토리지 보관함을 탐색하여, 이미 제작했던 책을 다시 클릭하거나 재검색했을 경우 **API 호출과 로딩창을 건너뛰고 0.1초 만에 캐시 데이터로부터 즉시 복원**하여 책을 펼쳐 줍니다.

## 5. CORS 보안과 Vercel 라우팅 우회 (Rewrites)
웹 프로그래밍을 처음 시작할 때 가장 많이 겪는 장벽 중 하나가 바로 **CORS(Cross-Origin Resource Sharing)** 보안 오류입니다.

* **CORS 예방**:
  `api/index.py` 내부에 FastAPI CORS 미들웨어를 장착하여 외부 브라우저의 접근을 허용하고, `vercel.json`의 라우팅 매핑 지시문들을 통해 동일 도메인 상에서 백엔드와 프론트엔드가 자연스럽게 어우러지도록 설계했습니다.

---

## 6. 동적 Blob & 가상 Object URL을 이용한 파일 내보내기 (Export)
우리가 작성한 책을 개인 컴퓨터로 다운로드받는 기능은 **바이너리 파일 생성 및 DOM 트리거 기술**로 구현되었습니다.

1. **HTML 텍스트 문서 가공**: `js/app.js`에서 현재 열람 중인 도서 스토리와 일러스트 이미지 링크를 취합해 인쇄용 미려한 스타일이 내장된 HTML 문자열을 작성합니다.
2. **Blob 객체 생성**: 자바스크립트 메모리상에 텍스트 데이터를 컴퓨터가 이진수로 읽을 수 있는 파일 데이터인 **Blob(Binary Large Object)** 객체로 포장합니다.
3. **가상 URL 발급 및 강제 클릭**: `URL.createObjectURL`로 가상 경로를 연결하고 임시 `<a>` 링크의 `.click()` 이벤트를 강제로 터뜨려 파일 저장을 유도합니다.
